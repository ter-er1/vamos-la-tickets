📋 PROBLEMAS IDENTIFICADOS E CORRIGIDOS
================================================================================

1. ❌ ANDROIDMANIFEST.XML - PERMISSÕES FALTANDO
   Problema: O arquivo não tinha permissões de CÂMERA, INTERNET e NETWORK
   Sintoma: Compilação falhava ou app crasheava ao iniciar
   Solução: Adicionadas as 3 permissões obrigatórias
   Status: ✅ CORRIGIDO

2. ❌ GRADLE.PROPERTIES - CONFIGURAÇÕES OBSOLETAS
   Problema: 
   - android.enableR8=false desabilita minificação (causa bloat de build)
   - android.useLegacyToolchain=true incompatível com Gradle 8+
   - JVM memory: -Xmx1g era insuficiente
   Sintoma: Build travava ou falhava por razões obscuras
   Solução: 
   - Removidas flags obsoletas
   - Aumentado JVM memory para -Xmx2g
   - Adicionado android.ndkVersion
   Status: ✅ CORRIGIDO

3. ❌ SETTINGS.GRADLE.KTS - LOCAL.PROPERTIES OBRIGATÓRIO
   Problema: Script tentava ler local.properties que não existe no CI/CD
   Sintoma: GitHub Actions não conseguia localizar Flutter SDK
   Solução: Simplificado para usar configuração padrão do Gradle + Flutter
   Status: ✅ CORRIGIDO

4. ❌ BUILD.GRADLE.KTS - REFERÊNCIAS INVÁLIDAS
   Problema: Código referenciava flutter.* que não estavam disponíveis
   Sintoma: Gradle falhava durante parsing do arquivo
   Solução: Substituídas por valores fixos e estáveis
   Status: ✅ CORRIGIDO

5. ❌ WORKFLOW - TRATAMENTO DE ERRO INSUFICIENTE
   Problema: Mensagens de erro genéricas, logs não capturavam detalhes
   Sintoma: Impossível diagnosticar erro real
   Solução: Workflow simplificado, adicionado verificação de APK
   Status: ✅ CORRIGIDO

🎯 POSSÍVEIS ERROS FUTUROS (E COMO EVITAR)
================================================================================

ERRO 1: "Cannot locate Flutter SDK in FLUTTER_HOME"
├─ Causa: GitHub Actions não consegue encontrar Flutter
├─ Prevenção: ✅ Já corrigido no settings.gradle.kts
└─ Solução: Usar ação subosito/flutter-action (que já fazemos)

ERRO 2: "Out of memory during Gradle build"
├─ Causa: JVM RAM insuficiente
├─ Prevenção: ✅ Aumentamos -Xmx2g e desabilitamos parallelismo
└─ Solução: Se acontecer, aumentar para -Xmx3g

ERRO 3: "Permission denied" ao usar câmera
├─ Causa: App não tem permissão declarada
├─ Prevenção: ✅ Adicionamos permissions no AndroidManifest
└─ Solução: Adicionar mais se necessário

ERRO 4: "minSdk = X is higher than targetSdk = Y"
├─ Causa: Versões do SDK inconsistentes
├─ Prevenção: ✅ Configuramos minSdk=21, targetSdk=35 consistentemente
└─ Solução: Manter ambos iguais

ERRO 5: "Cannot download dependency from Maven Central"
├─ Causa: Rede instável ou repositório indisponível
├─ Prevenção: GitHub Actions usa rede estável (raro acontecer)
└─ Solução: Retry automático no workflow

ERRO 6: "Namespace conflicts"
├─ Causa: namespace declarado em 2 lugares (build.gradle + AndroidManifest)
├─ Prevenção: ✅ Usamos apenas em build.gradle (Android Gradle Plugin 8+)
└─ Solução: Remover namespace do AndroidManifest se der erro
