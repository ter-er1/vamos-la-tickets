🔄 FLUXO DE VALIDAÇÃO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Scanner lê QR Code
   └─ Contém: ticket_id + event_id + timestamp + signature (HMAC-SHA256)

2. App valida localmente (estrutura JSON)
   └─ Se inválido: mostra erro (vermelho)

3. App envia para Servidor via HTTP/LAN
   └─ IP: 192.168.1.100:8000
   └─ Latência: ~50ms

4. Servidor valida em 4 passos:
   ├─ 1️⃣ Verifica assinatura HMAC-SHA256
   ├─ 2️⃣ Busca ticket no banco
   ├─ 3️⃣ Verifica se já foi usado
   └─ 4️⃣ Marca como "used" (lock atômico)

5. Servidor responde:
   ├─ ✅ "valid" → App mostra verde + beep sucesso
   ├─ ⚠️ "already_used" → App mostra amarelo + beep erro
   └─ ❌ "invalid" → App mostra vermelho + beep erro


🔐 SEGURANÇA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Assinatura HMAC-SHA256:
  message = ticket_id + event_id + timestamp
  signature = HMAC_SHA256(message, SECRET_KEY)

Por que é seguro:
  ✅ SECRET_KEY apenas no servidor (app não conhece)
  ✅ Sem SECRET_KEY = impossível falsificar QR
  ✅ Timestamp valida - rejeita QRs antigos
  ✅ Qualquer modificação invalida a assinatura

Contra fraude:
  ✅ Lock atômico previne duplicação
  ✅ Não confia no app (validação no servidor)
  ✅ Cada ticket único (UUID)
  ✅ Todos scanners verificam contra servidor

💻 SERVER LOCAL (Node.js)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Stack:
  🔹 Express.js (framework web)
  🔹 SQLite3 (banco de dados)
  🔹 HMAC-SHA256 (criptografia nativa do Node)
  🔹 CORS (aceita requests da LAN)

Endpoints:
  POST /validate-ticket       ← Principal, chamado 100+ vezes/evento
  POST /sync-tickets          ← Sincronizar tickets em bulk
  POST /events                ← Criar novo evento
  GET  /events                ← Listar eventos
  GET  /events/:id/stats      ← Estatísticas
  GET  /validation-logs/:id   ← Histórico
  GET  /health                ← Health check

Características:
  ✅ Lock atômico (SQL)
  ✅ Transações ACID
  ✅ Índices otimizados
  ✅ Sem dependências externas (crypto nativo)
  ✅ Pronto para rodar 24/7

   ├─ No router: Reservar IP para PC
   └─ Todos scanners usarão este IP

2. Atualizar IP no App
   ├─ Arquivo: mobile_app/lib/main.dart
   └─ Linha ~16: const String SERVER_URL = 'http://192.168.1.100:8000'

3. Atualizar SECRET_KEY (produção)
   ├─ Arquivo: local_server/.env
   └─ Chave forte (ex: 32 caracteres aleatórios)

4. Criar evento antes do evento
   ├─ Via API: POST /events
   └─ Sincronizar tickets: POST /sync-tickets
