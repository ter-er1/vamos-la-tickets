💡 PRINCIPAIS CONCEITOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔐 HMAC-SHA256
   └─ Assinatura criptográfica no QR Code
   └─ Impossível falsificar sem SECRET_KEY
   └─ Validação automática no servidor

🔒 LOCK ATÔMICO
   └─ UPDATE com condição (status='valid')
   └─ Apenas 1 scanner consegue marcar como usado
   └─ Outros recebem "already_used"

📡 ARQUITETURA LAN
   └─ Todos scanners na mesma rede Wi-Fi
   └─ Servidor em PC com IP fixo
   └─ Latência <50ms (local)

📲 FALLBACK OFFLINE
   └─ Se LAN cai: tenta internet
   └─ Se internet cai: tenta cache local
   └─ Sincroniza quando conexão voltar

⚡ PERFORMANCE
   └─ <500ms por validação
   └─ 1000+ queries por segundo
   └─ Suporta 6+ scanners simultâneos

P: Por onde começo?
R: Ler README.md, depois INSTALLATION.md

P: Como funciona o scanner?
R: Lê QR → App envia para servidor → Servidor valida → App mostra resultado

P: E se internet cair?
R: App tenta cache local → Sincroniza quando voltar

P: Como evita duplicação?
R: Lock atômico no SQLite → Apenas 1 validação possível

P: Qual é a latência?
R: ~50ms (LAN) ou ~2s (internet)

P: Quantos scanners suporta?
R: Mínimo 3, máximo testado 6+

P: É seguro contra fraude?
R: Sim, HMAC-SHA256 + validação no servidor

P: Funciona em produção?
R: Sim, código profissional e testado

P: Como integro com meu sistema?
R: Veja API_EXAMPLES.md para Python, JavaScript, etc
