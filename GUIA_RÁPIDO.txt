ARQUIVO DE IP IMPORTANTE:
═══════════════════════════════════════════════════════════════════════════════

Antes de compilar o app, você PRECISA atualizar o IP do servidor!

Arquivo: mobile_app/lib/main.dart
Linha:   ~16

Procure por:
  const String SERVER_URL = 'http://192.168.1.100:8000';

Mude para seu IP:
  const String SERVER_URL = 'http://192.168.1.50:8000';
  (trocar 192.168.1.50 pelo seu IP real)

Como descobrir seu IP:
  Windows: ipconfig
  Linux:   ifconfig
  (procurar por "IPv4 Address" ou "inet")

PROBLEMAS COMUNS:
═══════════════════════════════════════════════════════════════════════════════

Problema: "npm: command not found"
Solução:  Instalar Node.js em https://nodejs.org

Problema: "flutter: command not found"
Solução:  Instalar Flutter em https://flutter.dev

Problema: "Porta 8000 em uso"
Solução:  
  Windows: netstat -ano | findstr :8000
           taskkill /PID <PID> /F
  Linux:   lsof -i :8000
           kill -9 <PID>

Problema: "App não conecta ao servidor"
Solução:  
  1. Verificar IP em mobile_app/lib/main.dart
  2. Verificar se servidor está rodando (npm start)
  3. Verificar se está na mesma rede WiFi
  4. Testar ping: ping 192.168.1.100

Problema: "Testes falhando"
Solução:  
  1. Verificar se servidor está rodando
  2. Executar novamente: node test-api.js
  3. Reconstruir database: rm database/tickets.db
